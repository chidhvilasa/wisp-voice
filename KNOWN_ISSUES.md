# Known Issues

This is an honest list of the current limitations of Wisp. None of these block the v0.1.0 frontend/test pipeline, but all of them matter before treating a build as production-ready.

- **Rust toolchain required for a full build.** The frontend (this repo's `npm run build`/`vitest`) builds and tests without Rust, but producing an actual installable binary (`npm run tauri build`) requires a working Rust stable toolchain and the platform-specific Tauri prerequisites. This environment does not have Rust installed, so the Tauri build itself has not been run or verified end-to-end yet — only the CI/release workflows have been authored to do so on GitHub's runners.
- **`rnnoise-wasm` is a stub, not real noise suppression.** The noise-suppression AudioWorklet currently wraps a placeholder package rather than a compiled RNNoise WASM binary. Noise suppression toggles in Settings will not meaningfully reduce background noise until this is replaced with a real WASM build.
- **Soundboard clips have a practical ~2MB size limit.** Soundboard files are base64-encoded and transferred/stored as strings rather than streamed as binary, which works for short clips but will visibly degrade (slow encode, large persisted state) well before any "real" audio file size.
- **Overlay drag/snap is untested at runtime.** The corner-snapping and drag-to-reposition logic in `Overlay.tsx` has only been exercised through unit/stress tests with mocked Tauri APIs; it has not yet been verified against a real Tauri window (real monitor geometry, real mouse events) because no Rust build has been run in this environment.
- **TURN is a free public relay, not production-scale.** The TURN fallback (`openrelay.metered.ca`) is suitable for testing and small-scale use but offers no uptime guarantee, bandwidth guarantee, or abuse protection. A production deployment should provision a dedicated TURN server (e.g. coturn on your own infrastructure, or a paid TURN provider).
- **Hotkey re-registration may fail silently on some Linux window managers.** Global shortcut registration goes through `tauri-plugin-global-shortcut`, which depends on the window manager's support for global key grabs. On some Linux WMs (particularly tiling WMs with their own keybinding layers), re-registering a hotkey after the user changes it in Settings may fail without a visible error to the user.
